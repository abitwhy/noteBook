css预处理器 Less 和 Sass 的区别
Less 和 Sass 的共同点 序号 Less 的特点 Sass 的特点
  1. 混入(Mixins)——class 中的 class;
  2. 参数混入——可以传递参数的 class，就像函数一样;
  3. 嵌套规则——class 中嵌套 class，从而减少重复代码;
  4. 运算——CSS 中进行算数运算（包括函数方法）;
  5. 颜色功能——可以编辑颜色;
  6. 命名空间（namespace）——分组样式，实现相互调用;
  7. 作用域——局部修改样式;
  8. JavaScript 赋值——在 CSS 中使用 JavaScript 表达式赋值;
 1 Less 环境较 Sass 简单，只需要引入 Less.js 编译 .less 文件即可，也可以在开发环境实用工具先编译好。 Sass 的变量有作用域，有全局和局部变量之分，并且有优先级。
2 Less 使用较 Sass 简单，没有裁剪 CSS 原有特性，仅为 CSS 添加了程式语言的特性，容易上手。 Sass 有函数的概念，更接近程式化编程（混入（minxin） 类似函数，但缺乏编程逻辑，不过目前使用得最广泛）。
3 通过客户端动态编译时，速度相对较慢。 通过服务端动态编译时，速度相对较快。
4 能够进程控制，如条件：@if、@else；循环：@for、@each、@while；继承：@extend、引用：@import。
5 拥有数据结构，如数组：$list、对象：$map、以及 string、number 等。
