## css属性分类的思考

*大部分 css 属性的分类工作，其实会在使用不同的选择器阶段，就已不知不觉地做了粗分类，以下只针对同一选择器下仍出现的众多复杂属性做分类。*

### 定义

关于 css 属性分类方式：（有先后顺序，今后也遵照此顺序写属性）

1. 直观与否

   - 直观，属性值发生改动，能立即在元素渲染上直观显示出来，且具有完全可控性。如宽高，外边距，背景颜色等属性。它们的改动最为频繁，需要先行设置（写在前面）。

     1.1 表现分类

     - 外观
     - 布局 

   - 不直观，属性值发生改动，在元素渲染上不能明显示出来，或者不具有完全可控性。如盒子模型（`box-sizing`），浮动（`float`），展示（`display`）。他们的改动不多，但设置顺序两极化，有的最先设置，有的最后调整阶段时设置

     ，统一把它们写在后面。

### 例子

也就是说，以后的 css 代码分出三部分，如下

```css
* {
    /* 直观-外观 */
    width:123px;
        
	/* 直观-布局 */
    margin: 0 auto;
    
    /* 不直观 */
    float:left;
}
```
